# 🚀 cascade

Каскадный скрипт для проброса входящего трафика с одного сервера на другой.

---

## 📌 Описание

`cascade` — это простой bash-скрипт, который настраивает перенаправление всего входящего TCP/UDP трафика с одного сервера (точка входа) на другой (основной сервер).

Исключения:
- `22` — SSH  
- `80` — HTTP  
- `443` — HTTPS  

Это позволяет:
- сохранить доступ к серверу по SSH  
- использовать веб-сервер на входной ноде  
- скрыть основной сервер обработки трафика  
- разделить инфраструктуру на вход и backend  

---

## 🧠 Как это работает

1. Клиент подключается к первому серверу  
2. Первый сервер принимает трафик  
3. Через iptables (DNAT) трафик пересылается на второй сервер  
4. Второй сервер обрабатывает соединение  

📍 В результате:
- снаружи виден только первый сервер  
- фактическая обработка происходит на втором  

---

## 🖥️ Схема
`
[ Клиент ]`
`↓`
`[ Сервер #1 (Entry / Cascade) ]`
`↓`
`[ Сервер #2 (Backend / VPN) ]`

---

## 🌍 Пример реальной схемы

Пример использования:

- **Сервер #1 (точка входа)**  
  Хостинг: aeza  
  Локация: Санкт-Петербург  

- **Сервер #2 (основной сервер)**  
  Хостинг: phylex.net  
  Локация: Европа  

📌 Как это выглядит:
- пользователь подключается к серверу в РФ  
- весь трафик автоматически уходит на европейский сервер  
- VPN или сервисы работают уже на втором сервере  

---

## ⚙️ Установка и использование

### 1. Подготовка

- Арендуйте 2 сервера  
- Убедитесь, что есть root-доступ  

---

### 2. Настройка backend-сервера

На втором сервере:
- установите VPN (например: AmeziaWG)  
- убедитесь, что он работает  

---

### 3. Запуск cascade

На первом сервере:

```bash
wget https://raw.githubusercontent.com/IGrok2/cascad/main/main.sh
chmod +x main.sh
./main.sh
```
